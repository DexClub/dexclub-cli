## Guidance

- For `usingStrings`, prefer `Contains` with `ignoreCase: true` unless exact matching is explicitly required.
- For class, method, field, and type names, `Equals` is usually the better default.
- When the user asks which methods call a known target method, prefer `invokeMethods` on candidate callers.
- Use `searchPackages` before adding deeply nested matcher branches when the target package is known.
